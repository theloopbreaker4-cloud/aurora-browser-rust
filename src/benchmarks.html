Engine Benchmarks

External tests that probe what the active engine actually supports. Use these to spot gaps that aurora://test (which only checks API surface) can miss — especially layout/rendering correctness and real-world JS performance.
How to read results: on Servo many of these will fail or score low — that is the point. They show concrete features that need work upstream (stylo / WebRender / Servo's script crate). Run the same benchmark on the WebView2 engine to see how things should look, and on Servo to see where we are.

Standards Compliance

html5test.com
html5test.com
Aggregate score 0-555 covering every HTML5 / DOM / forms / media / location feature. Best single number for "how complete is this engine".
Spec coverage
Web Platform Tests
wpt.live
Official W3C/WHATWG conformance suite. ~2 million tests written by spec authors. Drill into any subsystem (CSS, DOM, fetch, ...) and see exact pass/fail rates.
Official spec
Acid3
acid3.acidtests.org
Classic 100-point CSS / SVG / DOM / ECMAScript test. Old but quick — good smoke test that nothing fundamental is broken.
Smoke test
Can I Use
caniuse.com
Per-feature support matrix. Search any web feature and see which engines (and versions) ship it. Useful while triaging "why doesn't this site work".

Performance

Speedometer 3
browserbench.org/Speedometer3.0
Industry-standard SPA benchmark. Runs identical TodoMVC-style apps in React, Vue, Angular, Svelte, Preact, Lit, vanilla JS, etc. and measures runs/min. The closest thing to "how does this engine feel on a real app".
SPA perf
MotionMark
browserbench.org/MotionMark1.3
Stress test for the renderer: animated SVG, paths, leaves, design, text, images, suits. Each subtest scales complexity until FPS drops below 50. Output is a perceptual score.
Render perf
jsbench.dev
jsbench.dev
Quick microbenchmarks for arbitrary JS snippets. Useful when chasing a specific perf regression in Servo's script engine.
JS micro
JetStream 2.2
browserbench.org/JetStream2.2
Pure JS / WASM throughput & latency benchmark. Includes startup time which matters for short-lived workloads.
JS engine

Aurora Built-in

Aurora Engine Tests
aurora://test
Local feature-detection suite. Catches regressions when an API stops being exposed. Does NOT prove the API actually works correctly — for that, use the external benchmarks above.
